# Standby Cover Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add a hidden, toggleable full-screen **Standby Cover** to the OBS **Stint** scene that hides the live feeds **and** the POV PiP while staying **below the HUD** (Race Control banner + timer stay visible), via an idempotent maintainer script.

**Architecture:** A new idempotent script `tools/add_standby_cover.py` mutates the tokenized scene collection `src/obs/IRO_Endurance.json` — mirroring the existing `tools/add_pov_source.py`. It (a) appends an `image_source` named `Standby Cover` pointing at the already-tokenized `__IRO_ASSETS__/YT-IRO-Race.png`, (b) moves the existing `Feed POV` scene item from front-most to directly after `Feed B`, and (c) inserts a hidden full-screen `Standby Cover` scene item directly after `Feed POV` — both sit below the HUD group in z-order. A stdlib test (`tests/test_standby.py`, same runnable-script style as `tests/test_pov.py`) drives the mutation function on a synthetic collection. Docs get an incident cue (Director guide) and a technical note (OBS & scenes).

**Tech Stack:** Python 3 stdlib only (`copy`, `json`, `sys`, `importlib`). No new runtime deps. OBS scene-collection JSON (OBS 30 schema).

## Background the engineer needs

**OBS scene-collection JSON shape (only what matters here):**
- Top-level `d["sources"]` is a flat list of every source *and* every scene/group. A
  scene has `"id": "scene"`; a group has `"id": "group"`; an image is `"id": "image_source"`.
- A scene's items live in `scene["settings"]["items"]`, ordered **back → front** (last =
  front-most / on top).
- The **HUD is a group** (`name": "HUD"`, `"id": "group"`). In the Stint scene its three
  children are duplicated as scene items carrying `"group_item_backup": true` (`Overlay`,
  `HUD Overlay`, `HUD Race Timer`); the group itself renders at the position of the
  **non-backup** `HUD` item. **Treat `group_item_backup: true` items as inert** — never
  move them, never count them when deciding z-order. Only non-backup items define the
  render stack.
- Current Stint non-backup order (back → front): `Discord`, `Feed A`, `Feed B`,
  `HUD` (group), `Standings`, `Season Schedule`, `Race Results`, `Quali Results`,
  `Feed POV`. After this change it becomes: `Discord`, `Feed A`, `Feed B`, **`Feed POV`**,
  **`Standby Cover`**, `HUD` (group), `Standings`, `Season Schedule`, `Race Results`,
  `Quali Results`.
- A full-screen image insert (template: the `Standings` scene item) uses `pos {0,0}`,
  `scale {1,1}`, `bounds_type: 2` (OBS "fit" / `OBS_BOUNDS_SCALE_INNER`), `bounds
  {1920,1080}`.

**Key facts (copy verbatim):**
- New source name: `Standby Cover`; new uuid: `bbbbbbb1-0000-4000-8000-000000000001`
  (the A/B/POV feeds use the `aaaaaaaN-…` range; this picks a fresh `bbbbbbb1-…`).
- Image file token: `__IRO_ASSETS__/YT-IRO-Race.png` (same file the `Thumbnail` source uses).
- Template source to deep-copy: `Thumbnail` (an `image_source` with the same file).
- Template scene item to deep-copy: `Standings` (the full-screen image insert transform).
- The collection MUST stay tokenized (`__IRO_ASSETS__/…`) so `tools/build.py` verify passes.

**How to run the existing tests** (each test file is a runnable script — no pytest):
```bash
python3 tests/test_pov.py      # existing, must still pass
python3 tests/test_standby.py  # new, this plan
```

```python
#!/usr/bin/env python3
"""Add the 'Standby Cover' incident hold graphic to an OBS scene collection (idempotent).

Mirrors tools/add_pov_source.py. It deep-copies the existing 'Thumbnail' image source
(same YT-IRO-Race.png file) and the 'Standings' full-screen scene item as templates, so
the result always matches OBS's schema. The cover sits BELOW the HUD group (Race Control
banner + timer stay visible); the existing 'Feed POV' item is moved below the HUD too
(directly after Feed B) so the cover hides the POV PiP as well. Re-running is a no-op
once 'Standby Cover' exists.

Usage: python3 tools/add_standby_cover.py <collection.json>
"""
import copy, json, sys

COVER_NAME = "Standby Cover"
COVER_UUID = "bbbbbbb1-0000-4000-8000-000000000001"   # A/B/POV use aaaaaaaN-; cover bbbbbbb1-
COVER_FILE = "__IRO_ASSETS__/YT-IRO-Race.png"          # reuse the standby graphic (tokenized)
POV_NAME   = "Feed POV"


def _feedb_index(items):
    """Index of the non-backup 'Feed B' scene item."""
    return next(i for i, it in enumerate(items)
                if it.get("name") == "Feed B" and not it.get("group_item_backup"))


def add_standby_cover(d):
    """Mutate the collection dict in place. Return True if changed, False if already present."""
    srcs = d["sources"]
    if any(s.get("name") == COVER_NAME for s in srcs):
        return False
    thumb = next(s for s in srcs if s.get("name") == "Thumbnail")
    stint = next(s for s in srcs if s.get("id") == "scene" and s.get("name") == "Stint")
    items = stint["settings"]["items"]

    # 1) source object: copy Thumbnail (same image_source/file), override identity
    src = copy.deepcopy(thumb)
    src["name"] = COVER_NAME
    src["uuid"] = COVER_UUID
    src["settings"] = dict(src.get("settings", {}))
    src["settings"]["file"] = COVER_FILE
    srcs.append(src)

    # 2) move existing 'Feed POV' (if present) to sit directly after Feed B
    insert_at = _feedb_index(items) + 1
    pov = next((it for it in items if it.get("name") == POV_NAME
                and not it.get("group_item_backup")), None)
    if pov is not None:
        items.remove(pov)
        insert_at = _feedb_index(items) + 1       # recompute after removal
        items.insert(insert_at, pov)
        insert_at += 1                            # cover goes after the moved POV

    # 3) Stint scene item for the cover: copy the 'Standings' full-screen insert as template
    tmpl = next(it for it in items if it.get("name") == "Standings")
    item = copy.deepcopy(tmpl)
    item["name"] = COVER_NAME
    item["source_uuid"] = COVER_UUID
    item["visible"] = False
    item["locked"] = False
    item["id"] = max(it.get("id", 0) for it in items) + 1
    item["pos"] = {"x": 0.0, "y": 0.0}
    item["scale"] = {"x": 1.0, "y": 1.0}
    item["bounds_type"] = 2                        # OBS_BOUNDS_SCALE_INNER = "fit"
    item["bounds"] = {"x": 1920.0, "y": 1080.0}
    items.insert(insert_at, item)
    return True


def main(path):
    with open(path, encoding="utf-8") as fh:
        d = json.load(fh)
    if not add_standby_cover(d):
        print(f"{path}: '{COVER_NAME}' already present — skip"); return
    with open(path, "w", encoding="utf-8") as fh:
        json.dump(d, fh, ensure_ascii=False, indent=4)
    print(f"{path}: added '{COVER_NAME}' (hidden full-screen, below HUD) "
          f"and moved '{POV_NAME}' below the HUD group")


if __name__ == "__main__":
    if len(sys.argv) != 2:
        sys.exit("usage: add_standby_cover.py <collection.json>")
    main(sys.argv[1])
```
