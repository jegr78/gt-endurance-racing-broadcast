The relay reads its schedule and HUD data from the Google Sheet and is configured through Settings: RACECAST_SHEET_ID (the sheet) and RACECAST_SHEET_PUSH_URL (write-back for the timer + panel). There are no other local relay settings — tab names, ports and the bind mode use safe defaults.

Fonts & advanced CSS

The builder writes the generated profiles/<active>/overlay/<page>.css plus a layout-<page>.json it owns. Save writes both; Apply in OBS reloads the HUD/Timer browser sources (needs the relay up + obs-websocket). First override on a profile that had no overlay yet? Restart the relay once (racecast relay restart) to activate it; later edits apply live via Apply in OBS. The canvas shows sample data over your Overlay.png frame — verify live with Preview.

Looks

named overlay + graphics + media snapshots

A Look backup snapshots the active profile's overlay CSS, graphics and media into a zip. Restore overwrites the current files and reloads OBS browser sources (graphics and media reload on the next scene activation). Backups are stored in runtime/<profile>/backups/.

    A curated set of free Google Fonts ships with every install and is offered in the Overlay Builder's font pickers. Add any other family by name above — it is downloaded once into runtime/fonts/.
